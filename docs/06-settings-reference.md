# Settings Reference

All project settings live in `settings.py` at the root of your project (generated by `rusjango new`).

## How settings are loaded

Settings are loaded by `rusjango.settings.load_settings`, which:

1. Reads the specified Python file (default: `settings.py`) using `importlib`.
2. Collects every name that is **fully uppercase** (e.g. `DEBUG`, `DATABASE`).
3. Returns them as a plain `dict`.

```python
# rusjango.settings (simplified)
import importlib.util

def load_settings(path: str = "settings.py") -> dict:
    spec = importlib.util.spec_from_file_location("settings", path)
    mod  = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(mod)
    return {k: v for k, v in vars(mod).items() if k.isupper()}
```

The resulting dict is injected into every request's ASGI scope at
`scope["rusjango"]["settings"]` so that middleware and handlers can read
configuration without importing global state.

## Settings table

| Name | Type | Default | Description |
|---|---|---|---|
| `APP_NAME` | `str` | `"myproject"` | Project name; used in admin panel titles and log output. |
| `DEBUG` | `bool` | `True` | When `True`, full Python tracebacks are included in 500 error responses. Set to `False` in production. |
| `SECRET_KEY` | `str` | *(generated)* | Signing key for tokens and cookies. **Must be changed before deploying to production.** |
| `ALLOWED_HOSTS` | `list[str]` | `["*"]` | Accepted `Host` header values. Ignored when `DEBUG=True`. Supports `"*"` wildcard and `.example.com` subdomain patterns. |
| `INSTALLED_APPS` | `list[str]` | `[]` | Dotted paths to app packages. Each package must expose `router = Router()` inside its `api.py`. |
| `MIDDLEWARE` | `list[str]` | `["rusjango.security.SecurityMiddleware"]` | Dotted paths to ASGI middleware classes. Applied in **reverse order** — the last entry is the outermost wrapper. |
| `DATABASE` | `dict \| None` | `None` | Database connection config. Set to `None` to disable the ORM entirely. See details below. |
| `AUTH` | `dict \| None` | `None` | Authentication configuration. *(Phase 6 — not yet implemented.)* |
| `ADMIN` | `dict \| None` | `None` | Admin panel configuration. *(Phase 5 — not yet implemented.)* |
| `AI` | `dict \| None` | `None` | AI/LLM provider configuration. *(Phase 9 — not yet implemented.)* |
| `WORKER` | `dict \| None` | `None` | Background task worker configuration. *(Phase 8 — not yet implemented.)* |
| `PAYMENTS` | `dict \| None` | `None` | Payments provider configuration. *(Phase 10 — not yet implemented.)* |

## Setting details

### `APP_NAME`

```python
APP_NAME = "myproject"
```

A human-readable name for the project. Appears in:
- The admin panel page title (Phase 5).
- Structured log output.
- Error page headers.

---

### `DEBUG`

```python
DEBUG = True   # development
DEBUG = False  # production
```

- `True` — 500 responses include a full Python traceback in the JSON body under the key `"traceback"`.
- `False` — 500 responses return only `{"error": "Internal Server Error"}`.

Also controls whether `ALLOWED_HOSTS` is enforced (it is **not** enforced when `DEBUG=True`).

---

### `SECRET_KEY`

```python
SECRET_KEY = "change-me-in-production-use-a-long-random-string"
```

Used to sign:
- JWT tokens (Phase 6).
- Signed cookies.

Generate a strong key with:
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

> **Warning:** Never commit a production `SECRET_KEY` to version control.

---

### `ALLOWED_HOSTS`

```python
ALLOWED_HOSTS = ["localhost", "127.0.0.1"]            # exact match
ALLOWED_HOSTS = [".example.com"]                      # all subdomains
ALLOWED_HOSTS = ["api.example.com", ".example.com"]   # mixed
ALLOWED_HOSTS = ["*"]                                  # allow all (not recommended in prod)
```

Validated by `SecurityMiddleware` on every request when `DEBUG=False`. A request whose `Host` header does not match any entry receives a `400 Bad Request` response.

Pattern rules:
- `"*"` — matches any host.
- `".example.com"` — matches `example.com` and all subdomains (`api.example.com`, `www.example.com`, …).
- Any other string — exact, case-insensitive match.

---

### `INSTALLED_APPS`

```python
INSTALLED_APPS = [
    "apps.school",
    "apps.accounts",
]
```

Each entry is a dotted Python package path. Rusjango imports `<app>.api` and reads `router` from it. The router is mounted at `/api/<app_leaf_name>/` (e.g., `apps.school` → `/api/school/`).

Requirements per app package:
- Must be importable (i.e., on `sys.path`).
- Must contain `api.py`.
- `api.py` must define `router = Router()`.

Apps are added and removed automatically by `rusjango add app <name>` and `rusjango remove app <name>`.

---

### `MIDDLEWARE`

```python
MIDDLEWARE = [
    "rusjango.security.SecurityMiddleware",
    "myapp.middleware.LoggingMiddleware",
]
```

Middleware classes are applied in **reverse list order** when building the ASGI stack:

```
Request  →  LoggingMiddleware  →  SecurityMiddleware  →  Core handler
```

This matches Django's convention. See `docs/07-middleware.md` for details and how to write custom middleware.

### `DATABASE`

#### SQLite

```python
DATABASE = {
    "ENGINE": "sqlite",
    "NAME": "db.sqlite3",   # path relative to project root
    "ASYNC": True,          # uses aiosqlite
}
```

#### PostgreSQL

```python
DATABASE = {
    "ENGINE": "postgresql",
    "URL": "postgresql://user:pass@localhost:5432/dbname",
    "ASYNC": True,          # uses asyncpg connection pool
}
```

#### Disabled

```python
DATABASE = None
```

When `None`, the ORM is not initialised and no database connection is opened at startup. Accessing ORM methods while `DATABASE=None` raises `RuntimeError`.
